# Public Release Checklist

Use this before publishing the repository, enabling GitHub Pages, or creating a release.

## Repository Basics

- README explains the product in the first screen.
- LICENSE exists.
- SECURITY exists.
- CHANGELOG exists.
- CONTRIBUTING exists.
- Suggested topics are listed.
- Sample output is fictional or approved for public use.

## Safety

- No `.env` file.
- No API keys.
- No tokens.
- No cookies.
- No webhooks.
- No SSH keys.
- No private customer data.
- No internal pricing, margin, or contract details.
- No guaranteed ranking or indexing claims.

## GitHub Settings

- Repository visibility is intentional.
- Secret scanning / push protection is enabled when available.
- Default branch is `main`.
- GitHub Pages source is set to GitHub Actions.
- Branch protection or ruleset is enabled before team collaboration.

## Status Boundary

- Draft package prepared: `TECH_OK_UNPROVEN`.
- Published public repository with clean scan: `DONE_WITH_EVIDENCE`.
- Published Pages demo reachable: `DONE_WITH_EVIDENCE`.
- Real customer uses it and confirms value: `ACCEPTED`.
